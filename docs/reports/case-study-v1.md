# [Report] 동시성 제어 실무 도입 사례 통합 분석 (Case Study V1)

**작성일:** 2026-02-05
**목적:** 4가지 동시성 제어 방식이 실제 현업(Tech Giants)에서 어떻게 적용되었는지, 어떤 트레이드오프를 거쳐 선택되었는지 통합 분석한다.

---

## 1. Summary: "성능과 정합성의 줄다리기"

모든 아키텍처 결정은 **"얻는 것(Gain)"**과 **"잃는 것(Cost)"**의 교환입니다.

| 기업/서비스 | 도메인 | 선택한 방식 | **얻은 것 (Gain)** | **지불한 비용 (Cost)** |
| :--- | :--- | :--- | :--- | :--- |
| **금융권 (은행)** | 계좌 이체 | **Pessimistic Lock** | **완벽한 정합성 (Trust)** | 낮은 처리량, DB 스케일 아웃 불가 |
| **Notion** | 문서 협업 | **Optimistic Lock** | **실시간 동시성 (UX)** | 충돌 시 재시도 구현 복잡도 |
| **토스 (Toss)** | 결제 시스템 | **Redis Dist. Lock** | **DB 보호 (Throttling)** | Redis 운영 비용, 인프라 복잡도 |
| **배달의민족** | 선착순 쿠폰 | **Lua Script** | **압도적 속도 (TPS)** | 비동기 복구(Reconciliation) 구현 비용 |

---

## 2. 상세 사례 분석 (Deep Dive)

### 🏦 Case 1: 금융권 & 커머스 결제 (Pessimistic Lock)
*   **상황:** 내 통장 잔고가 100만 원인데, 동시에 100만 원 이체를 2번 시도함.
*   **핵심 가치:** "마이너스 통장이 되는 것보다, 차라리 1초 늦게 이체되는 게 낫다."
*   **선택:** `SELECT ... FOR UPDATE`로 데이터베이스 레벨에서 강력하게 잠금.
*   **Lesson:** 돈 문제 앞에서 성능 타령하지 마라. **ACID가 최우선**이다.

### 📝 Case 2: Notion & Confluence (Optimistic Lock)
*   **상황:** 수십 명이 동시에 문서를 보고 편집함.
*   **핵심 가치:** "편집할 때마다 락 걸려서 멈추면 아무도 안 쓴다."
*   **선택:** 락 없이 읽게 하고(`No Lock`), 저장할 때만 버전(`version`)을 확인.
*   **Lesson:** **충돌(Conflict)을 에러가 아니라 '비즈니스 흐름'**으로 인정하고, UX(자동 병합 등)로 풀어내라.

### 🛡️ Case 3: 토스 & 쿠팡 (Redis Distributed Lock)
*   **상황:** MSA 환경. 주문 서버와 결제 서버가 다름. 트래픽 폭주로 DB가 죽기 직전.
*   **핵심 가치:** "DB는 소중하다. Redis를 방패막이로 세워라."
*   **선택:** Redis 분산 락으로 유입량을 제어(Throttling)하고, DB는 유니크 키로 최종 방어만 함.
*   **Lesson:** Redis 락은 단순한 상호 배제가 아니라, **시스템 전체를 보호하는 '관문(Gatekeeper)'**이다.

### 🏎️ Case 4: 배민 & 티켓링크 (Lua Script)
*   **상황:** 아이유 콘서트 티켓팅. 1초에 10만 명 접속.
*   **핵심 가치:** "0.001초라도 더 빨라야 한다. 락 거는 시간도 아깝다."
*   **선택:** Redis 내부에서 `GET + SET`을 원자적(Atomic)으로 실행하는 Lua Script 사용.
*   **Lesson:** 극한의 성능이 필요할 땐, **애플리케이션 로직을 데이터 저장소(Redis) 안으로 밀어넣어라(Compute near Data).**

## 3. 결론: "비즈니스가 기술을 정의한다"

기술적으로 "가장 빠른 락"은 Lua Script지만, 모든 곳에 Lua를 쓰지 않습니다.
*   **은행**은 느려도 안전한 **비관적 락**을 쓰고,
*   **위키**는 락 없는 **낙관적 락**을 씁니다.

**"우리 서비스는 무엇을 팔고 있는가? 신뢰인가, 속도인가, 경험인가?"**
이 질문에 대한 답이 곧 사용할 기술을 결정합니다.
